# GTA7 — Design Spec

A GTA-style 3D game in Rust + Raylib. Procedural city, driving, on-foot, shooting,
peds/traffic/cops, wanted system, health/armor, money/pickups/shops, missions, HUD.
Zero external assets — all textures and models generated in code.

## Stack
- Rust 1.95, `raylib` crate v6.0 (safe bindings).
- Single binary, single crate. No external assets.
- Procedural textures via `GenImage*` + `LoadTextureFromImage`.
- Models composed from primitives in code (boxes, cylinders, capsules, planes).

## Timestep & Loop
- **Fixed-timestep logic** at a selectable rate: **30 / 60 / 90 / 120 Hz** (default 60).
- Hotkey cycles the rate (`F2`); current rate shown in a debug overlay (`F1` toggles overlay).
- **Render decoupled** from logic: render every vsync frame, vsync on.
- **State interpolation**: accumulator with alpha = accumulated_time / fixed_dt.
  Render uses `prev_state.lerp(cur_state, alpha)` for entity transforms (player, vehicles,
  peds, cops, camera). Keeps 30Hz logic fluid and 120Hz crisp without variable-rate integration.
- Input sampled per render frame; fed into logic as discrete events + held-state snapshot.

## World — procedural grid city
- Seeded RNG → N×N block grid. Each block: sidewalks + buildings (varied height/color),
  with parks and empty lots mixed in. Roads between blocks, lane markings, crosswalks,
  traffic lights at intersections.
- Ground plane. Buildings = axis-aligned boxes.
- Collision: circle-vs-AABB resolution (player/vehicles vs buildings), ray-vs-box (hitscan),
  capsule-vs-box (melee/ped). Simple, fast, robust.
- Lane graph = grid edges (directed lanes). Traffic and cops path along it.

## Camera — third-person follow
- On foot: orbit camera (mouse look), player capsule below pivot.
- In vehicle: chase camera lagging, tracking velocity direction.
- Smoothed; collision pull-in to avoid clipping buildings.
- Interpolated via the same alpha as entities.

## Player — on-foot + driving
- Capsule character: move (WASD), sprint (Shift), jump (Space), mouse aim.
- Press `F` near a car to enter/exit. Driving takes over controls; on-foot disabled in vehicle.
- Health 100, armor 0–100. Inventory: unarmed, pistol (start), bought weapons.

## Vehicles — arcade physics
- Box car body + 4 visual wheels. Arcade model: forward accel, friction,
  steering proportional to speed, handbrake (Space) for drift, gravity + ground snap.
  Body rotates; velocity aligns to heading gradually.
- Collisions: vs buildings = bounce + damage; car-vs-car = impulse exchange + damage.
- Damage reduces a car HP; destroyed cars explode (particle + damage radius) and become wreckage.
- Traffic cars follow lane graph, stop at red lights, crude avoidance of player car.

## Combat — shooting
- Pistol hitscan: muzzle flash sprite, tracer line, recoil kick, reload (`R`), ammo.
- Hitscan vs peds/cops/cars: ray-vs-box/capsule. Blood particle; ped ragdoll-falls (tip + fade).
- Melee (`F` unarmed): short-range knockback + small damage.
- Player can be shot by cops (wanted ≥2) — uses same hitscan against player.

## NPCs — peds + cops
- Peds wander sidewalks, flee when gunfire nearby, drop cash on death, ragdoll.
- Cops spawn at wanted ≥1: chase on foot and in cars, shoot at wanted ≥2,
  roadblocks at wanted ≥4. Up to 6 stars with escalating counts.

## Wanted system
- Stars 0–6. Crimes add heat: kill ped → +threshold, kill cop → +2 stars worth,
  destroy car → +1. Heat decays over time when out of cop line-of-sight.
- HUD shows stars.

## Health/armor/damage
- Player HP 100, armor 0–100 (absorbs first). Fall damage, crash damage, bullets.
- Death → respawn at nearest hospital, lose a cash fraction (GTA-style), wanted cleared.

## Money + pickups + shops
- Cash from ped drops, missions, stunts. HUD money counter.
- Pickups: health (heart), armor (vest), money, weapon/ammo.
- Shops: walk into marker, press `E` to buy weapon/health.

## Missions
- Rotating objective loop: pink marker → reach → sub-objective (kill target / deliver car /
  survive X sec) → reward. Simple state machine. Mission text banner in HUD.

## HUD (raylib shapes/text)
- Top-right: wanted stars, money, weapon + ammo.
- Bottom-left: health + armor bars.
- Bottom-right: minimap (top-down: roads, player arrow, blips).
- Mission text banner (top-center). Crosshair in combat.

## Architecture / modules
```
src/
  main.rs            — window, loop, fixed timestep + interpolation, state dispatch
  config.rs          — settings (logic rate, debug, bindings), hotkey handling
  game.rs            — Game state, update orchestration, spawn/wanted logic
  time.rs            — accumulator, alpha, rate switching
  world/
    city.rs          — procgen grid, buildings, roads, lane graph, traffic lights
    collision.rs     — circle/AABB, ray/box, capsule/box, resolve
  player.rs          — on-foot controller, health, inventory
  vehicle.rs         — car physics + enter/exit + damage
  camera.rs          — follow/orbit cameras, interpolation
  combat.rs          — hitscan, tracers, particles, melee
  ai/
    ped.rs           — pedestrian state machine
    cop.rs           — cop + police car AI
    traffic.rs       — traffic lane following
  wanted.rs          — heat system
  pickup.rs          — pickups + shops
  mission.rs         — objective system
  hud.rs             — HUD + minimap
  render/
    models.rs        — procedural meshes/textures (car, building, char, ground, road)
    fx.rs            — particles, tracers, flashes, explosions
  input.rs           — input mapping + bindings
  mathx.rs           — helpers (lerp, angle wrap, easing)
```

## Testing
- Unit tests (pure logic fns, `cargo test`):
  - collision resolution (circle/AABB push-out, ray/box hit + normal).
  - wanted heat math (add/decay/star thresholds).
  - vehicle integration step (one tick at known input → expected velocity/heading).
  - mission state transitions.
  - procgen determinism (same seed → identical city layout).
- No mocks. Render/integration verified by running the binary.

## Scope & risk
- Procedural models = zero asset pipeline; risk is visual blandness, mitigated with
  varied palette, generated window textures, one directional light + ambient.
- Arcade (not rigid-body) physics keeps vehicle code small and stable; no physics dep.
- Single crate; fast compile.
